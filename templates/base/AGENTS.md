# __STACK_DISPLAY_NAME__

This project is a Bun workspaces monorepo generated by __STACK_DISPLAY_NAME__.

## Structure

- `frontend`: Vite + React app with TanStack Router, TanStack Query, Tailwind, and example frontend tests
- `backend`: Hono API with Drizzle, PostgreSQL Docker setup, and example backend tests
- `shared`: shared types, Zod schemas, constants, utilities, and example shared tests
- `mosskit.json`: manifest describing which optional MossKit features are enabled for this app

## Directory Layout

- `frontend/src/app`: frontend providers and app wiring
- `frontend/src/routes`: TanStack Router route files
- `frontend/src/components`: starter UI components and frontend example tests
- `frontend/src/utils`: API and env helpers
- `backend/src/routes`: Hono route modules
- `backend/src/db`: Drizzle schema and database wiring
- `backend/src/utils`: backend helpers such as env parsing and auth utilities
- `shared/src/types`: shared TypeScript types used across frontend and backend
- `shared/src/validation`: shared Zod schemas and validation helpers
- `shared/src/utils`: shared runtime-safe utilities

## How To Work In This App

- Shared contracts and validation should live in `shared` first, then be consumed from frontend and backend.
- Keep route handlers close to their Hono route definitions rather than introducing a controller layer.
- Root scripts coordinate common tasks such as `dev`, `build`, `test`, `lint`, `typecheck`, and database commands.
- `bun run dev` starts both frontend and backend from the repo root.
- `bun run db:start` and `bun run db:stop` manage the local PostgreSQL container.
- Optional features should be reflected in `mosskit.json`; use the MossKit CLI to inspect or add them.

## Verification

- `bun run test`: runs backend, frontend, and shared example tests
- `bun run typecheck`: runs TypeScript checks across all workspaces
- `bun run build`: builds backend and frontend
- `bun run lint`: runs Oxlint across the repo

## Notes

- Deployment is intentionally left to the app owner.
- Auth and shadcn/ui are optional starter features and may or may not be present.
