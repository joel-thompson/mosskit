# MossKit

MossKit is a Bun-first starter generator for a full-stack monorepo built around Vite + React on the frontend, Hono on the backend, and a shared TypeScript package for contracts, validation, and reusable code.

## Structure

- `packages/create-mosskit`: the CLI that scaffolds new apps
- `templates/base`: the default generated app
- `templates/features/auth`: Clerk auth overlay applied by the CLI
- `templates/features/shadcn`: shadcn/ui overlay applied by the CLI
- `tests`: smoke tests for the generator
- generated apps include a root `mosskit.json` manifest used by `info`, `features`, and `add`

## How This Repo Works

- The public package is `@joelthompson/create-mosskit`.
- The CLI logic lives under `packages/create-mosskit/src`.
- The package copies from `templates/base` first, then applies overlays from `templates/features/*`.
- The packaged `templates/` directory under `packages/create-mosskit` is generated by `node ./scripts/sync-templates.mjs`.
- New projects are expected to be created and managed with `bunx @joelthompson/create-mosskit ...`.

## Working On The Generator

- If you change scaffold logic, README generation, package metadata handling, or feature application, check `packages/create-mosskit/src/scaffold.js`, `src/index.js`, `src/readme.js`, `src/features.js`, and `src/manifest.js`.
- If you change template files, update `templates/base` or `templates/features/*` first, then run `node ./scripts/sync-templates.mjs`.
- If you change dependency versions for generated apps, edit `packages/create-mosskit/src/dependency-versions.js` and run `node ./scripts/sync-template-package-versions.mjs`.
- The generated app verifier is the real end-to-end check. Use it after dependency changes, template changes, or CLI behavior changes.

## Verification

- `node --test tests/*.test.mjs`: smoke tests for generator shape, manifests, feature overlays, and expected files
- `node ./scripts/verify-generated-apps.mjs`: scaffolds fresh apps and runs install, dev, typecheck, test, and build
- `npm pack --dry-run --cache /tmp/mosskit-npm-cache` from `packages/create-mosskit`: checks the publishable package surface

## Releases

- Publish from `packages/create-mosskit`, not the repo root.
- The package is scoped, so publishing should use `npm publish --access public`.
- The `prepack` hook syncs packaged templates before packing or publishing.
- The local release checklist is kept in `RELEASE_PROCESS.local.md`, which is gitignored on purpose.

## Generated App Shape

Generated projects use:

- `frontend`: Vite, React, TanStack Router, TanStack Query, Tailwind
- `backend`: Hono, Drizzle, PostgreSQL, Docker Compose
- `shared`: shared types, Zod schemas, constants, and utilities
